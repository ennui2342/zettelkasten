# Foundations of LLM-Based Autonomous Agents: Reasoning, Tool Use, and Planning

LLM-driven agents have become a central paradigm for complex, multi-step tasks across software development, research, operations, legal, and enterprise domains. An agent combines a reasoning LLM, memory, and tools, operating within a Partially Observable Markov Decision Process: perception converts observations into representations, memory maintains mutable state, a cognitive step produces a reasoning trace, and an action policy selects operations. Environmental feedback closes the loop. Real-world workflows exceed single model calls because they demand persistent state, iterative reasoning, and external action beyond any single context window.

### Structured Reasoning

Prompting techniques unlock latent reasoning across a complexity spectrum. **Chain-of-Thought (CoT)** decomposes reasoning into sequential steps; **self-consistency** samples multiple chains and votes on answers. **Tree-of-Thought (ToT)** enables branching and backtracking; **Graph-of-Thought (GoT)** generalizes further with directed graphs supporting loops, refinement, and aggregation. Specialized variants include program-of-thought (executable code for numerical tasks) and algorithm-of-thought (broader exploration in single invocations). Effectiveness is task-dependent: simple problems need basic CoT; challenging mathematical reasoning benefits from ToT/GoT; direct-retrieval tasks can be hindered by explanation-heavy prompts.

The **ReAct paradigm** interleaves reasoning traces with actions, improving groundedness but remaining vulnerable to error propagation where early mistakes compound into unproductive loops. Which reasoning structure to embed is an architectural decision shaping the agent's competence profile.

### Planning and Search

Agents leverage probabilistic world knowledge to propose and revise plans, connecting goals to executable action sequences. Plans can be represented as DAGs with preconditions, intermediate states, and temporal dependencies—their structure determines what context to preserve at each step. Dedicated planning components (Planner-LMs) explore this but typically do not optimize the interaction between plans and the context upon which planning depends.

**Language Agent Tree Search (LATS)** connects planning to Monte Carlo Tree Search, using the model as both proposal generator and evaluator—improving decisions at higher computation cost. Frontier reasoning models increasingly internalize search at inference time with configurable "thinking" budgets, shifting external planning toward safety enforcement, state persistence, and tool permissions.

**Hierarchical decomposition** addresses context and modularity limits through recursive sub-agent architectures and milestone-planner/executor structures, improving interpretability and fault isolation at higher token cost.

**Reflection and self-correction** allow agents to critique and adapt from trajectories—storing failure critiques for cross-episode learning, running generate-critic-revise loops, and extracting reusable rules from mistakes. Purely internal reflection risks confirmation bias; tool-interactive validation mitigates this by grounding critiques in external evidence.

### Orchestration

A major trend favors controllable orchestration over maximizing autonomy. Graph-based frameworks model workflows as state machines with cycles, conditional branching, checkpointing, and human approval gates. Agentic systems exhibit variability in execution paths, tool selection, and memory retrieval due to inherent non-determinism, causing uncertainty to propagate across tool invocation, memory consistency, and multi-agent interactions.

### Memory

Memory mitigates context-length limitations. Inspired by human cognitive structures—working, episodic, semantic, and procedural memory—researchers have designed agent memory stacks. Systems like Reflexion store self-assessments; Generative Agents maintain episodic memories and synthesize reflections; MemAgent uses reinforcement learning to manage fixed-size memory. State-of-the-art approaches extract entity relations into graph or vector-based engines, supporting single-hop retrieval, multi-hop reasoning across interconnected instances, temporal reasoning, and open-domain retrieval combining conversational memory with external knowledge.

Agent context decomposes into actionable, structural, referential, and justificatory components. Most memory systems emphasize storage and retrieval but do not optimize the shape or content of context presented during reasoning. RAG improves information access but retrieved content is typically appended rather than shaped or compressed, and tool outputs accumulate without distillation—the relevance dilution problem persists even with 128k–1M+ token windows, where attention degrades over large inputs. Architectural approaches (BigBird, LongT5, Compressive Transformers) and context-control methods (COMPASS, FoldGRPO, ReCAP, GoA) are complementary to context-level optimization at the agent layer.

### Tool Use and Action Interfaces

Tool use separates autonomous agents from bare language models. Challenges span selecting correct tools, populating parameters accurately, and adapting to interface changes. Modular tool definitions isolate faults and support independent validation; a robust pattern treats tools as secure microservices decoupled from reasoning logic. Retrieval relevance is often limited to a single upcoming query, and tool outputs accumulate without distillation.

The action space spans five paradigms:

**API-centric systems** (Toolformer, Gorilla, MRKL, PAL) emphasize safety through structured function calls. The **Model Context Protocol (MCP)** standardizes tool discovery and invocation with governance boundaries including allowlists, authentication, and audit logging.

**Code-as-action** uses executable scripts as the control interface. CodeAct demonstrates superior performance; Voyager's reusable skill library shows agents composing complex behaviors; Code-as-Policies demonstrated LLMs reasoning about long-horizon physical tasks through programming abstractions.

**Agent-computer interfaces** provide curated, agent-friendly environments—SWE-agent reduces context load via simplified shells designed for agent interaction.

**Computer-use actions** enable generic GUI operation through screenshots with mouse/keyboard inputs (Anthropic's computer-use, OpenAI's Operator), broadening reachable tasks but expanding the attack surface for indirect prompt injection.

**Embodied VLA models** push toward direct perception-to-motor control. SayCan used LLMs as planners filtered through affordance functions; Gemini Robotics introduces native thinking for robots with cross-embodiment transfer; VLM-GroNav integrates proprioceptive sensing with vision to detect hazards vision alone misses.

### Observability, Safety, and Risk

Deploying agents in consequential settings demands transparency. Observability platforms log every reasoning step, tool call, and decision linkage. Human-in-the-loop escalation flags novel or high-stakes situations. Protocols like **Agent-to-Agent (A2A)** enable capability discovery via Agent Cards, supporting evaluation where auditor agents assess behaviors through tool logs, memory queries, and state changes. When agents execute actions, hallucinations become concrete failures; indirect prompt injection remains a central security concern—amplified by computer-use modes.

### Limitations of Single-Agent Systems

Long-horizon tasks strain context windows and degrade coherence. Diverse skill requirements exceed what a single persona reliably handles. Growing tool libraries deteriorate action selection. These limitations motivate multi-agent architectures distributing problems among role-specialized agents with independent reasoning, tool access, and domain focus.
