## Hub Notes

A small set of notes accumulated repeated updates, becoming synthesis points for the corpus. These emerged without pre-planning — retrieval kept finding them as the most relevant landing spot for incoming content.

| Note | Self updates | Cross-paper updates | From papers | Final size |
|------|-------------|---------------------|------------|------------|
| Memory Poisoning, Multi-Agent Security & Broader Safety Landscape | 3 | 6 | 17, 18 | 21,571 chars |
| Multi-Agent LLM Systems: Architecture, Memory & Collaborative Reasoning | 1 | 5 | 10, 14, 15, 18 | 21,095 chars |
| Benchmarking Agentic AI Systems: The AgentArch Framework | 3 | 5 | 11, 12 | 21,708 chars |
| Retrieval-Augmented Generation over Graph-Structured Data | 2 | 3 | 6 | 19,452 chars |

**Self updates** are UPDATEs from the same paper that created the note — the creating paper generated multiple drafts that all converged on the new note. These reflect topic richness within a single paper, not retrieval working across papers.

**Cross-paper updates** are UPDATEs from later papers that retrieved the note independently. This is the genuine hub behaviour — retrieval found the same note as the most relevant target for content from different source papers. Memory Poisoning drew 6 updates from two late, dense papers (17, 18); AgentArch drew 5 from papers 11 and 12.

Multi-Agent LLM Systems drew the widest spread of cross-paper updates (papers 10, 14, 15, 18) and was also the first note created in the run. Two mechanisms are in play and cannot be cleanly separated in a single sequential run: (1) co-activation weight accumulated from every cluster it appeared in, compounding retrieval score over time; (2) notes created first in an empty store are forced to be broad and foundational — Form has no existing neighbourhood to refine against, so early drafts capture the most general framing available. Later papers can be more specific because the broad concepts are already there. The wide cross-paper spread may therefore reflect genuine semantic centrality rather than a retrieval artefact — both mechanisms point in the same direction. This mirrors citation network dynamics: foundational papers accumulate citations not because they remain the most definitive treatment but because they named the concept first and the domain's vocabulary radiates outward from them. A potential failure mode for the benchmark: hub notes may be over-retrieved for questions where the most relevant answer lives in a later, more specific note that co-activation has not yet elevated.

Note also that hub note titles evolved across updates — AgentArch's note was renamed mid-run from "The AgentArch Framework for Evaluating Multi-Agent Architectures in Enterprise Settings" to "Evaluating Multi-Agent Architectures Across Enterprise and Academic Settings", reflecting genuine synthesis rather than accumulation.

**Single-paper convergence** (not a hub): Plan-Aware Context Engineering for LLM Agents was created in paper 20 (PAACE) and received all 5 of its updates within that same paper's batch (04:01–04:08). All updates are self-referential. Final size 20,723 chars reflects depth within one paper, not cross-paper synthesis. Removed from hub list.

## Notable Observations

### Zero NOTHING decisions

The system never decided a draft was too marginal to integrate. Combined with sparse STUB usage (3 operations), the pipeline has a strong bias toward inclusion. This is appropriate for knowledge synthesis (avoid losing information), but means the benchmark corpus may include some notes whose connection to the broader graph is weak. Worth monitoring as the store grows.

- "Memory Poisoning Attacks on LLM Agents"

Neither too broad (not "Multi-Agent Systems") nor too narrow (not paper summaries). The Form phase is producing well-scoped concepts that survive as stable integration targets.
