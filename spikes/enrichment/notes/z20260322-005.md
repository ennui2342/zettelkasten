# Experience-Driven and Self-Evolving AI Systems

A promising approach to adaptive decision-making in AI systems involves accumulating and leveraging past execution experience to continuously improve future behavior. This paradigm centers on a **self-evolving experience base**—a structured knowledge repository that logs execution data at a fine-grained (step-level) granularity. Each record captures multi-faceted information including the agent role, the LLM used, the sub-task instruction (with its embedding for semantic retrieval), tools invoked, monetary cost, wall-clock duration, execution success, and task-level success rate. After each task execution, newly generated records are appended, allowing the knowledge base to grow into a rich repository capturing the interplay between sub-task characteristics, model choices, and outcomes.

**Cold start** is a fundamental challenge for experience-based systems, as an initially empty knowledge base renders retrieval and routing mechanisms ineffective. This can be addressed through dedicated exploration strategies—for example, executing a curated set of agentic tasks with stochastic model selection (uniform random sampling from the model pool) to populate the experience base with diverse initial data. Such initialization can be accomplished with modest cost and time, yielding sufficient step-level records to provide an informative prior.

**Multi-faceted retrieval** enables the system to identify historically analogous sub-task executions when a new sub-task arises. Rather than relying on a single matching criterion, retrieval aggregates records matching on any of several facets: agent role matching (functionally equivalent precedents), semantic similarity retrieval (using pre-trained sentence encoders like MiniLM with cosine similarity thresholds), and tool congruence retrieval (matching based on predicted tool usage overlap). The union of these independently retrieved subsets ensures a broad and varied pool of evidence for decision-making.

**Tool prediction** for retrieval purposes can employ a two-stage hybrid strategy: a near-instantaneous heuristic check using keyword-to-tool dictionaries, with fallback to a lightweight LLM call for ambiguous cases, balancing accuracy with minimal computational overhead.

### From Retrieved Evidence to Model Selection: Pareto Filtration and Thompson Sampling

Once multi-faceted retrieval has assembled a pool of historical records for a given sub-task, the system faces a multi-objective selection problem: choosing among candidate LLMs that differ along competing axes of performance, cost, and efficiency. These objectives cannot all be simultaneously maximized, so the selection mechanism must navigate trade-offs rather than collapse them into a single score.

**Pareto filtration** provides the first stage of pruning. A candidate model is dominated if another model is superior or equal on all three trilemma axes and strictly superior on at least one. Retaining only non-dominated models forms the Pareto-optimal set, eliminating clearly inferior options while preserving the diversity of trade-off solutions. This efficiency-focused pruning step reduces the decision space without discarding any model that offers a genuinely distinct trade-off profile.

Selecting greedily from the Pareto set, however, would cause stagnation—the system would repeatedly exploit the currently best-looking model and never discover that an uncertain alternative might be superior. **Thompson sampling** addresses this as an exploration-exploitation mechanism. Each trilemma metric for each candidate model is treated as a continuous variable drawn from a Normal distribution, with uncertainty over its mean and variance modeled using a **Normal-Inverse-Gamma (NIG) conjugate prior**. Posterior distributions are dynamically constructed from the retrieved evidence (sample count, mean, and variance of each metric), and at decision time, stochastic samples are drawn from each model's posterior for each metric. The model with the highest sampled weighted utility is selected, where the weights reflect desired trade-off priorities (e.g., prioritizing performance while penalizing cost and delay).

This Bayesian approach naturally balances exploration of uncertain models—whose posteriors are wide due to sparse evidence—with exploitation of known good performers whose posteriors have tightened around favorable values. Ablation studies confirm that both components contribute meaningfully: replacing Thompson sampling with greedy selection reduces performance, while omitting Pareto filtration yields smaller but still notable degradation. The combination ensures principled exploration alongside efficiency-focused pruning.

### Dual-Phase Operation and the Self-Evolving Loop

The self-evolving nature of such systems is realized through dual-phase operation: an **optimization phase** that engages in tree-based exploration, sampling multiple trajectories to proactively diversify the knowledge base, and an **inference phase** that leverages accumulated wisdom to pursue a single optimized execution path for rapid, cost-effective task completion. During the optimization phase, Thompson sampling's exploratory tendency is especially valuable—uncertain models are tried, their outcomes recorded, and the posteriors updated, so that subsequent decisions rest on richer empirical ground. During the inference phase, posteriors have narrowed sufficiently that Thompson sampling's stochastic draws converge toward exploitation, and Pareto filtration ensures only genuinely competitive models are considered.

What distinguishes this paradigm from static procedural memory is the tight coupling between the experience base and the routing decision. Where procedural memory stores successful execution traces to guide future behavior of a fixed model, a self-evolving experience base stores outcome-annotated records across multiple models and configurations, enabling the system to learn not just *how* to execute a sub-task but *which model should execute it* and *at what expected cost and reliability*. The experience base thus serves simultaneously as a memory system and as the empirical foundation for dynamic model routing—collapsing what are traditionally separate concerns into a single, continuously enriched data structure.

This dual function has direct implications for the agent system trilemma. The optimization phase deliberately trades short-term cost and efficiency for long-term knowledge acquisition, exploring diverse model assignments to map the performance–cost landscape. The inference phase then exploits this map—via Pareto filtration and Thompson sampling over tightened posteriors—to select execution paths that are Pareto-optimal with respect to the current task's constraints. Over time, as the experience base grows, the system's effective trilemma frontier expands: tasks that once required expensive frontier models can be routed to cheaper alternatives whose adequacy has been empirically verified, while genuinely difficult sub-tasks are still escalated to high-capability models. The experience base thus functions as an amortization mechanism—prior exploration expenditure is converted into a reusable asset that subsidizes future routing precision.

The multi-faceted retrieval design also reflects a principled response to the heterogeneity of agentic workflows. No single similarity metric reliably captures when two sub-tasks will benefit from the same model assignment: two sub-tasks may share an agent role but require different tools, or invoke the same tools in semantically distinct contexts. By taking the union of role-matched, semantically similar, and tool-congruent records, the retrieval mechanism hedges against the failure of any single facet, producing a candidate pool that is robust to the varied dimensions along which sub-task similarity manifests. This is analogous to ensemble methods in prediction—diversity in retrieval facets reduces the variance of the evidence base, leading to more stable routing decisions. The Pareto filtration and Thompson sampling stages then operate over this robust evidence pool, ensuring that the final model selection is both well-informed and appropriately exploratory.
