# Automatic Workflow Optimization and Search

Reducing human effort in constructing multi-agent workflows has motivated several complementary research directions: prompt optimization, communication and topology optimization, and automatic workflow search and generation.

**Prompt optimization** approaches such as DSPy formalize modular prompt programs with compilation-time optimization, while TextGrad proposes gradient-inspired textual improvements. **Topology optimization** methods include GPTSwarm (graph-structured agent teams with iterative refinement), Dylan (dynamic agent selection), Agent-Prune (pruning redundant communication edges), and G-Designer (learning task-adaptive topologies). **Automatic workflow design** encompasses ADAS (heuristic search over agent architectures), AFlow (Monte Carlo Tree Search with named nodes), MaAS (optimizing architecture distributions via a controller), AgentSquare, and more recent methods that generate workflows through fine-tuning, continuous optimization, or reinforcement learning (MAS-GPT, FlowReasoner, ScoreFlow). Some approaches even train weaker meta-agents to design workflows for stronger executor models.

### Cost-Aware Search: The OneFlow Paradigm

OneFlow exemplifies a cost-aware automatic workflow design algorithm that formulates workflow optimization as a multi-objective problem balancing task performance and computational cost. It searches a discrete space of agents and communication patterns using Monte Carlo Tree Search (MCTS). The search process begins with the simplest possible workflow—direct input-output—as the root node, then iteratively applies four MCTS stages: **selection** (choosing a workflow from a candidate set using a mixed-probability strategy combining exploration and exploitation), **expansion** (generating new workflow candidates), **evaluation** (testing on validation data), and **backpropagation** (recording performance comparisons to avoid redundant designs).

A distinctive feature of this approach is its **dual meta-LLM architecture** during the expansion phase. A "Creative Designer" meta-LLM proposes performance-focused workflow improvements based on current workflow code, performance metrics, and detailed error logs from failure cases. A "Critical Reviewer" meta-LLM then refines these proposals by examining code comments, comparing against existing candidate workflows' cost and performance metrics, and optimizing for cost-efficiency. The reviewer's output becomes the final workflow for each iteration. This collaborative design explicitly balances the performance-cost trade-off. The resulting workflows tend to be more compact—featuring longer, more comprehensive system prompts for individual agents and fewer total agents—making them particularly suitable for single-agent execution.

### The Homogeneity Assumption and Its Implications

The vast majority of automatic design systems assume homogeneous executors, where all agents share the same base LLM and differ only in prompts, tools, and workflow position. Heterogeneous options—where agents use different base LLMs—are explored only tangentially. This default toward homogeneity is partly pragmatic: heterogeneous workflows dramatically expand the design space, requiring joint optimization over model selection, prompt engineering, tool assignment, and routing logic.

The strong performance of single-LLM baselines against automatically designed homogeneous workflows establishes an important reference point. Because all agents in a homogeneous workflow possess identical reasoning capabilities, a single agent can in principle simulate the entire workflow through sequential role-playing. This means automatic multi-agent design methods should demonstrably outperform this simpler single-agent alternative to justify their added complexity. When they fail to do so, the value proposition shifts from multi-agent coordination toward better prompt engineering and workflow structure within a single-agent paradigm—a conclusion reinforced by the compact, prompt-heavy workflows that cost-aware search methods tend to discover.

The genuine frontier for automatic workflow design likely lies in heterogeneous composition, where different models are assigned to different roles based on capability-cost profiles. This is where automatic search could provide value that manual design or single-agent simulation cannot easily replicate—but it requires search algorithms that can navigate the combinatorial explosion of model-role assignments alongside structural and prompt optimization.
