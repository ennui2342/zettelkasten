Title: TWICE: An LLM Agent Framework for Simulating Personalized User Tweeting Behavior with Long-term Temporal Features
Authors: Bingrui Jin, Kunyao Lan, Mengyue Wu
Published: 2025-12-15
Source: https://arxiv.org/abs/2602.22222
ArXiv ID: 2602.22222

TWICE: An LLM Agent Framework for Simulating Personalized User Tweeting Behavior with Long-term Temporal Features
Abstract
User simulators are often used to generate large amounts of data for various tasks such as generation, training, and evaluation. However, existing approaches concentrate on collective behaviors or interactive systems, struggling with tasks that require modeling temporal characteristics. To address this limitation, we propose TWICE, an LLM-based framework that leverages the long-term temporal and personalized features of social media data. This framework integrates personalized user profiling, an event-driven memory module, and a workflow for personalized style rewriting, enabling simulation of personalized user tweeting behavior while capturing long-term temporal characteristics. In addition, we conduct a comprehensive evaluation with a focus on analyzing tweeting style and event-based changes in behavior. Experiment results demonstrate that our framework improves personalized user simulation by effectively incorporating temporal dynamics, providing a robust solution for long-term behavior tracking.
1 Introduction
Current systems increasingly aspire to understand, emulate, and act for humans, realistic user simulation is paramount. Success in domains ranging from virtual assistants to dialogue agents and social simulation hinges on models’ capacity to simulate nuanced user behavior and expression with fidelity. However, authentic simulation demands far more than imitating discrete utterances or producing contextually appropriate outputs. Human behavior is highly individualized, affect-laden, and evolves over time (Chen2018WhatAM). For example, the same individual may respond quite differently to similar events at different life stages (Naslund2020SocialMA). Modeling such long-term behavioral dynamics remains a central challenge that most existing user-simulation approaches struggle to capture.
Although prior research has explored user simulation through interest modeling, dialogue strategies (article), and interaction feedback (louie-etal-2024-roleplay), these approaches often emphasize group-level patterns or short-horizon interactions while overlooking the “slow variables” of behavioral change: fluctuations in psychological state, the drift of linguistic style, and the residual effects of event memory. In other words, user simulation is no longer just about “what a user would say”, but “why the user would say it, and when.”
Among the various forms of user behavior, public posting offers distinctive research value. It naturally unfolds along a timeline, providing a window into how an individual’s expression evolves with events and context; it densely encodes idiolect and personality cues, serving as a stringent benchmark for testing whether a system can remain consistent amid change; and it motivates evaluation beyond purely functional outcomes toward a comprehensive suite that probes semantic fidelity, stylistic consistency, humanlikeness, and temporal alignment. Practically, posting simulation is transferable to applications such as personalized assistants and recommendation explanations, time-sensitive response generation, and interactions that are sensitive to mental-health signals. Thus, posting simulation is not merely a language-generation problem but a composite behavioral prediction task that integrates temporal, persona, and psychological modeling.
To this end, we formalize a concrete and consequential simulation task—tweeting behavior simulation—that, given a user’s history, current context, and event background, aims to generate the social media content the user is likely to produce. Building around this task, we propose twice (Temporal-dynamics–aware Workflow for Individualized, Context- and Event-driven user simulation), a user-simulation framework oriented toward long-term dynamics and personalization, and we design a systematic evaluation protocol spanning semantic alignment, emotional authenticity, stylistic fidelity, and temporal sensitivity.
We simulate personalized, temporally coherent posting by combining three purpose-built components. Profiles ground generation in stable user signals (demographics, traits, recurring themes), providing cleaner conditioning than raw timelines and reducing style drift. Event-driven memory restores causal and temporal coherence in dense, uneven histories by retrieving context using both time and semantics, improving continuity of emotional evolution. A two-stage workflow separates what to say (event-grounded content) from how this user would say it (personality- and style-guided rewriting), boosting stylistic fidelity without sacrificing content accuracy.
Our evaluation targets three research questions: RQ1 (Personalization)—profiles and style adaptation improve semantic and stylistic alignment; RQ2 (Temporal Alignment)—event-driven memory improves timeline coherence; RQ3 (Impact of Mental Health Status)-simulation quality between users with and without mental health disordrs. Metrics span consistency, authenticity, and humanlikeness. Across settings, the framework consistently improves semantic similarity, style similarity, and readability and aligns more closely with real behavior around temporal events, including same-user, different-time responses. In summary, the contributions of this paper are:
-
•
We identify user simulation with temporal change as a core challenge and formalize posting simulation as a temporally grounded, personalization-centric task.
-
•
We present TWICE, a model-agnostic framework that integrates personalized profiling, an event-driven memory balancing temporal locality and semantic relevance, and a two-stage style adaptation that separates content planning from persona-faithful expression; we also introduce a coherent evaluation protocol aligned with the above criteria.
-
•
The results demonstrate that our framework can achieve temporally coherent and personalized simulations with strong authenticity, consistency and humanlikeness, advancing user simulation from static to dynamic and from generic to personalized.

2.1 Personalized User Profiling
Accurately simulating what a user would say now requires more than a noisy, short-horizon timeline; it needs stable, identity-bearing priors that reduce ambiguity and mitigate style drift over months. We therefore construct a structured profile that the agent conditions on throughout generation. Conceptually, is a key–value store capturing both general facts and personalized regularities; operationally, it is designed to support personalization simulation by improving semantic and stylistic alignment with a user’s historical posts.
Definition and Scope
We represent as a collection of key–value pairs spanning demographics, work/marital status, Big Five personality traits, event-related features, and tweeting style. This balances breadth (coverage of who the user is) with density (high-information signals that remain predictive across time).
General attributes
For attributes such as age, gender, marital status, and career, we combine high-precision patterns with semantic matching to handle linguistic variability: multiple regular expressions maximize recall while preserving precision; a keyword-driven, embedding-based matcher computes cosine similarity between attribute-specific lexicons and tweet text to capture paraphrases; when fixed expressions require contextual reasoning, we apply lightweight LLM inference (e.g., ChatGPT-4o-mini) on the regex-matched spans to disambiguate. The overall extraction flow is illustrated in Fig. 2.
Personalized attributes
To encode slow, identity-level regularities and time-sensitive signals, we incorporate life events, symptom cues, Big Five traits, and stylistic evidence. Following the life-event taxonomy of Chen (Chen2024MappingLC) and the 38-category symptom schema of Zhang (Zhang2022SymptomIF), we reuse their supervised classifiers to obtain, for each tweet, an 11-dimension life-event score vector and a 38-dimension symptom relevance vector. We apply a hard threshold to retain only high-confidence hits, group tweets by detected event/symptom, and summarize each group with an LLM into a compact, information-dense snippet. In parallel, we infer Big Five traits from historical posts using LLM-based analyses. For style, we iteratively select representative posts across the timeline (e.g., review 100 tweets and keep 20 per iteration), and compress them into a concise natural-language style description plus an exemplar set. These ingredients jointly stabilize identity and furnish the downstream generator with precise cues for both “what to talk about” and “how this user tends to say it.”
Sampling
To generate diverse and realistic user profiles, we apply a density-aware sampling strategy based on Wang (Wang2025KnowYF). We embed 34,330 user profiles into a semantic space using SimCSE (Gao2021SimCSESC), incorporating both objective and subjective features. These embeddings are reduced via UMAP to preserve semantic structure, followed by Gaussian Kernel Density Estimation to model profile density. Guided by this density, we sample 977 representative profiles covering both common and rare cases, ensuring a balanced dataset for user simulation.
2.2 Event-driven Memory Module
User histories on social media are dense, uneven, and highly contextual; relying only on recency or surface similarity breaks causal chains and obscures how events shape expression over time. Our memory module is therefore event-centered and explicitly time-aware: it organizes the past so that generation conditions on the parts of history most likely to have produced the current utterance. This design targets temporal coherence while preserving semantic relevance and enabling analyses in RQ2/RQ3.
Memory Structure
We represent the historical context along two complementary axes. (i) General memory aggregates tweets into fixed windows (e.g., 30-day chunks, analogous to K6/K10 assessment periods) to capture short-term fluctuations; each tweet is embedded and pooled into a chunk-level representation. (ii) Event memory groups tweets by detected life events and symptoms to preserve high-signal, emotionally loaded trajectories; each event group is embedded to form a unified representation. The two views jointly capture when things happened and what they were about.

2.3 Model-Agnostic Workflow
We address how this user would speak now by decoupling content from expression and implementing a model-agnostic pipeline that works with any backbone LLM. The workflow comprises three tightly coupled steps—event extraction, content generation, and style adaptation—organized as two stages so that factual/semantic planning is separated from persona realization. This design aligns with our research questions: RQ1 (personalization efficacy, via profiles and style adaptation), RQ2 (temporal coherence, via event-aware retrieval), and RQ3 (stratified effects by mental-health status).
Event extraction
Using the LLM’s semantic analysis, we derive a compact event summary from recent posts or provided context, including type, affect, time/location expressions, and salient background (e.g., public issues). Aligning this summary with life-event/symptom signals in the profile ensures what happened and when are explicitly represented before generation, mitigating hallucinations and enforcing causal consistency.
Stage I: Event-grounded content generation
Conditioned on , the agent first produces an event-grounded draft that prioritizes semantic/factual alignment with the user’s past in similar contexts and respects the emotional trajectory implied by retrieved memories. This stage intentionally avoids overfitting to idiolect so that “what to say” is planned against a stable, high-information context.
Stage II: Style adaptation
The draft is then rewritten into the user’s voice without altering its semantics. The adaptation is guided by three signals already contained in : (i) Big Five cues (controlling stance, certainty/hedging, affect intensity), (ii) a concise natural-language style description (offering stable, controllable guidance across contexts), and (iii) a curated set of stylistic exemplars (iteratively selected historical posts that surface implicit habits such as indirectness, formulaic phrases, or preferred pacing). The result is output that says the same thing as this user would say it.
Separating content planning from stylistic realization gives fine-grained control over the trade-off between semantics and voice: Stage I anchors meaning; Stage II enforces idiolect. Together with profiling and event-driven memory, the workflow yields simulations that are simultaneously personalized and time-aware. In evaluation, we expect (and observe) higher semantic/style similarity and improved readability (FRE/FKGL), along with closer emotion-distribution proximity, while maintaining robustness across temporal configurations (§3).

3 Experiments
To evaluate how TWICE leverages personalized characteristics and long-term temporal patterns for authentic, consistent, and human-like tweet generation, we investigate:
-
•
RQ1 (Personalization Efficacy): To what extent do TWICE’s personalized profiles and two-stage style adaptation improve semantic and stylistic alignment with a user’s historical posts?
-
•
RQ2 (Temporal Coherence): How effectively does TWICE’s event-driven memory enhance timeline coherence by conditioning generation on relevant historical context?
-
•
RQ3 (Impact of Mental Health Status): How does simulation quality differ between users with and without mental health disorders, specifically regarding personalization and temporal coherence?
Dataset
Social media provides fine-grained traces of users’ momentary psychological states and their evolution over time (Chancellor2020MethodsIP; Ge2025ASO; Naslund2020SocialMA). We therefore use the Twitter-STMHD dataset (Suhavi2022TwitterSTMHDAE), which contains self-reported mental-health diagnoses alongside tweet text, social interactions, follower/following counts, and other longitudinal signals. For our experiments, we consider six diagnosis categories and one control group: ADHD (8,095), Anxiety (4,843), Bipolar (1,651), Depression (6,803), OCD (1,325), PTSD (3,414), and NEG—a control group without reported diagnoses (8,199). This corpus enables both between-group comparisons (RQ3) and within-diagnosed temporal analyses under varying configurations (RQ2), while supporting the personalization tests in RQ1.
Model Selection
In our experiments, we employed three different models for tweet generation: Qwen3-8B, Llama-3.1-8B-Instruct, and DeepSeek-R1-Distill-Qwen-7B. These models were selected to evaluate their performance in terms of various aspects such as semantic similarity, style, word overlap, and emotion intensity.
To isolate the effect of the backbone LLM, we keep all other variables fixed across models: identical prompts and decoding settings; the same TWICE pipeline (profile construction, event-driven memory, two-stage style workflow); fixed temporal parameters (e.g., time window, state coefficient, decay rate) and memory-retrieval settings (e.g., number of retrieved entries/top-, ranking weights); and identical user/event subsets and evaluation code. Only the LLM differs.

RQ1
Our goal is to reliably improve the personalization performance of user simulators in tweet generation tasks. To achieve this, we conduct a time-weighted event sampling strategy: events are first grouped by their tweet dates, then the proportion of events in each time segment is calculated as sampling weights. Sampling is performed based on these weights to ensure temporal representativeness of the selected events.
Settings Since the dataset spans different time periods, including the onset, diagnosis, progression stages of the illness, and the COVID-19 pandemic, time series are important here. We set temporal parameters as: time_window = one year; node_num = 3; memory_num = 10, 20; decay_lambda = 0.001; state_coeff = 1, 1.1. Note the impact of temporal parameters in memory retrieval on tweet simulation performance will be detailed in the next section. Based on this, we conduct large-scale repeated experiments using density-aware sampling of users and their experienced events. The experiment settings include:

RQ2&3
Although ordinary users also experience temporally-ordered events and fluctuations along different timelines, their event sequences often contain a high proportion of neutral events. These tend to have a limited impact on user states such as emotional variation. Furthermore, typical users generally exhibit more stable mental conditions, which may mask noticeable temporal variations. To better observe the ability of the Twice framework to capture such characteristics, we conduct experiments on users diagnosed with mental health disorders. These users are more likely to exhibit fluctuations in response to external stimuli, thereby enabling clearer observation of temporal changes.
Settings Given that the effectiveness of our framework in modeling personalized traits has already been validated in the previous section, we now focus on the temporal dimensions. Specifically, we examine the following parameters:time_window(the temporal range for memory retrieval), memory_num(the number of memory entries retrieved) and state_coeff(a factor controlling the retrieval probability of time-sensitive memory entries).
4 Results and Analysis
This section presents empirical results addressing TWICE’s performance across three dimensions: personalized alignment (RQ1), temporal coherence (RQ2), and mental status subgroup variations (RQ3). Quantitative analysis confirms significant improvements in simulation fidelity while highlighting temporal change as a critical performance factor.
RQ1 As shown in Table 3, the user simulator with both memory of event features and personalized user profiles (Memory=w/, Profile=event) consistently achieves the best performance across most evaluation metrics. This confirms the effectiveness of the TWICE framework in enhancing tweet simulation quality. The style rewriting workflow also contributes to stable improvements in semantic similarity, readability, and emotional consistency, indicating its value in generating more realistic user-like content. However, style similarity slightly drops after applying the workflow. We attribute this to the explicit reinforcement of user-specific language styles during rewriting, which may reduce naturalness in everyday tweets. Nonetheless, this enhancement could be beneficial in simulating reactions to more emotionally charged or significant events.
In addition, simulators equipped only with event memory (Memory=w/, Profile=-) show better performance in certain readability metrics. This may be due to the structured nature of the profile representation—formatted as key-value pairs—which could slightly interfere with the language model’s ability to mimic natural human expression. However, this impact appears limited and does not outweigh the benefits of incorporating user-specific information.
RQ2 We specifically analyzed the simulation trends of tweets for different users with mental health disorders under varying temporal parameters. Figure 3 presents four patients diagnosed with ADHD or Depression, each provided with general memory and event-feature memory for simulating tweet behavior. It can be observed that these four users exhibit consistent trends across parameters: the simulation quality improves as the time window expands, indicating that we can retrieve more comprehensive and deeper temporal information about the user’s timeline. This is particularly crucial for users with mental health disorders. Additionally, increasing the probability of retrieving memory entries containing event information leads to consistent improvements in tweet simulation for all four users, demonstrating the importance of event-based memory in tracking user behavior. Furthermore, increasing the number of retrieved memory entries within a reasonable range also enhances the simulation performance.
RQ3 We conduct extensive simulated posting experiments on both a user group with six types of mental health disorders (POS) and a control group with no mental health issues (NEG). We focused on the emotional simulation quality by examining the emotion KL divergence between simulated tweets and original tweets. As shown in the Table 4, regardless of the type of user profile or whether the rewriting workflow is applied, the simulated tweets for users with mental health disorders (Pos) consistently exhibit more accurate emotional representation. This highlights the strong performance of our framework in emotional tracking and reflects its capability to capture temporal characteristics.

Limitation
Currently, the quality of simulated tweets is primarily assessed using automated metrics such as semantic consistency and humanlikeness. Human evaluation has yet to be incorporated. Future research should introduce manual assessment to more comprehensively validate the authenticity and effectiveness of the generated content, particularly in terms of emotional depth and alignment with linguistic style.
5 Related Work
User simulation technology holds a significant position in the field of artificial intelligence. With the rapid advancement of LLMs, user simulation methods have evolved from early rule-based static modeling (Schatzmann2007AgendaBasedUS) and traditional data-driven approaches (Sutskever2014SequenceTS) to dynamic intelligent agent systems equipped with context-aware and personality-modeling capabilities (Wang2025KnowYF; Brown2020LanguageMA; Kong2023LargeLM; Wang2023UserBS). LLM-based user simulation not only excels in the naturalness and diversity of language generation but also demonstrates strong generalization abilities, enabling in-depth capture of user personality traits, emotional fluctuations, and behavioral patterns. This supports finer-grained and context-sensitive personalized simulation, significantly enhancing the realism and accuracy of simulations.
The applications of user simulation technology are extensive, spanning the training and evaluation of task-oriented dialogue systems (Kong2023LargeLM), user behavior generation in recommendation systems (Wang2023RecAgentAN), simulation of social behaviors in virtual social networks (Gao2023S3SS), and personalized interventions in digital mental health (Wang2025KnowYF). Particularly in scenarios involving personalized behavior modeling and ethically sensitive data processing, user simulation not only effectively reduces experimental and data collection costs but also provides a reliable technical alternative for related research. It promotes the intelligence and robustness of AI systems and drives technological innovation and application deployment across multiple domains.
6 Conclusion
In this paper, we propose TWICE, an LLM-based framework for simulating personalized user tweeting behavior with long-term temporal characteristics. By incorporating personalized user profiles, an event-driven memory module, and a style adjustment workflow, TWICE effectively captures both the individuality and temporal evolution of users in social media contexts. Through extensive experiments, we demonstrate that TWICE improves the authenticity, consistency, and emotional alignment of simulated tweets, especially in scenarios involving mental health-related users and temporally sensitive behavior. Our findings provide strong empirical evidence for the feasibility of simulating temporally evolving user behavior with LLM agents, paving the way for their application in personalized recommendation, affective computing, and digital mental health. This work highlights the potential of LLM-based simulators to bridge the gap between static modeling and dynamic user simulation, offering a new direction for research on user simulation and user-centric AI systems.
