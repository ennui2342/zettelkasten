# Labeled Property Graphs and Graph Query Languages

A Labeled Property Graph (LPG) is a labeled directed multigraph formally defined as a tuple consisting of a finite set of nodes, a finite set of edges (disjoint from nodes), a function mapping each edge to its source and target nodes, a partial labeling function assigning labels to nodes and edges, and a partial property function assigning key-value pairs to graph elements. Paths in a property graph are finite alternating sequences of nodes and edges, with path labels defined as the concatenation of edge labels along the traversal.

LPGs provide a more expressive data model than RDF triple stores, supporting rich attributes on both nodes and edges, flexible schemas, and native compatibility with declarative graph query languages. Graph databases built on the property graph model are NoSQL systems increasingly adopted across domains from bioscience to infrastructure for representing interlinked entities with rich semantics.

Cypher is the de facto standard query language for property graphs, originally developed for Neo4j and significantly influencing the ISO/IEC 39075:2024 GQL standard. Despite its widespread adoption in graph database systems, Cypher's application to LLM-backed graph reasoning and structured knowledge extraction has been limited compared to SPARQL's extensive use in semantic reasoning tasks. Text-to-Cypher—the task of translating natural language questions into executable Cypher queries—presents unique challenges compared to text-to-SQL, including handling multi-hop traversals, typed relationships, symmetric relationships, and compositional queries involving disjunctions or unions of structurally distinct subqueries.

Several benchmarks and datasets exist for evaluating text-to-Cypher systems. CypherBench is a benchmark built on Wikidata-derived property graphs spanning multiple domains with diverse query types aligned to realistic graph schemas, containing 7.8 million entities across 11 subject domains. SpCQL introduced the first large-scale text-to-Cypher dataset with 10,000 natural language–Cypher pairs. SynthCypher and SPIDER-Cypher address the lack of evaluation standards through automated pipeline generation and adapted benchmarks. Other datasets target specific domains such as transport networks and industrial maintenance.

Graph database backends for LPG querying include Neo4j and Memgraph, the latter being an in-memory, C++-backed, open-source graph engine optimized for low-latency query execution. Demonstrating compatibility across multiple LPG backends broadens the applicability of text-to-Cypher systems.
