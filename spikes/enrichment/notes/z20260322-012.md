# Multi-Agent Structured Query Generation and Self-Correction for Graph Databases

Translating natural language into executable queries over structured graph backends is a central challenge in GraphRAG systems. Multi-agent LLM architectures address this by distributing generation across specialized agents rather than relying on a monolithic model. Most GraphRAG work targets RDF knowledge graphs queried via SPARQL. Labeled Property Graphs (LPGs) queried via Cypher remain underexplored despite distinct advantages and challenges: greater schema variability, multi-hop traversal, typed relationships, and attribute-level reasoning—amplified when the system must construct correct, executable graph queries rather than retrieve text passages.

### Decomposition Patterns for Query Generation

The decomposition pattern is well-developed for text-to-SQL: frameworks use multiple LLMs for diverse candidate generation with chain-of-thought reasoning and pairwise selection; decomposer agents with auxiliary agents for sub-database retrieval and refinement; Monte Carlo Tree Search with self-supervised rewards for zero-shot generation; and earlier approaches including sketch-based slot filling, sequence-to-sequence models with reinforcement learning, relation-aware transformers for schema linking, bottom-up compositional parsing, and grammar-constrained decoding.

For text-to-Cypher, multi-agent pipelines instantiate analogous decomposition with graph-specific agents: query generation (grounded on schema formatted in Cypher-like syntax), named entity extraction, schema and entity verification, query execution against the graph backend, LLM-based evaluation (outputting structured feedback with discrete grades), instruction generation for corrections, and feedback aggregation.

### Verification, Self-Correction, and Hallucination Recovery

The verification module checks extracted entities via auxiliary Cypher queries. For unmatched entities, a two-step recovery retrieves candidates using normalized Levenshtein similarity, then uses an LLM to semantically rank alternatives. An instructions generator synthesizes revision guidance; a feedback aggregator consolidates evaluation and verification signals into prioritized corrections.

The iterative self-correction loop (typically up to four iterations) progressively improves queries by addressing misidentified entities, overly restrictive conditions, invalid traversal paths, incorrect relationship directions, and hallucinations. Effective strategies include reformulating equality checks into explicit value retrieval to make comparisons observable. This modular, database-grounded correction supports convergence toward schema-compliant queries. Related approaches include cycling through reasoning, interaction, and execution phases, and cooperative agentic exploration with multi-perspective self-reflection.

### Application to Building Information Modeling

Industry Foundation Classes (IFC), the open standard for BIM in architecture, engineering, and construction, encodes building elements, properties, spatial relationships, and hierarchical organization through an object-oriented structure with typed relationships. IFC data maps naturally to labeled property graphs, enabling Cypher-based querying and natural language interfaces to complex technical data without requiring expertise in the underlying format.

Evaluations on sample IFC building models show that multi-agent GraphRAG pipelines correctly answer questions about building components that simpler approaches fail to resolve, while expressing uncertainty when appropriate and grounding responses in graph database contents. This demonstrates how graph-based AI bridges complex engineering data standards and accessible information retrieval for industrial digital automation.

### Open Challenges in Structured Query Generation

On the query generation side, compositional queries requiring unions of structurally distinct subqueries, symmetric relationships complicating schema validation, and multi-intent questions requiring decomposition into independent subgoals remain difficult—potentially addressed through explicit query planning or intermediate symbolic representations. Future directions include larger domain-specific IFC-Cypher datasets and multi-turn dialogue for interactive building information exploration. Orchestration frameworks such as LangGraph support agent coordination and asynchronous state management across generation and verification stages.
