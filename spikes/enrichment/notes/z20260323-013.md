# Knowledge Distillation, Synthetic Data, and Plan-Aware Context Compression for Specialized Language Models

Training compact student models to replicate a larger teacher's behavior is a well-established paradigm, finding particular application in building efficient context compressors for agent systems. Understanding why simpler reduction techniques fall short requires situating distillation within the broader landscape of compression approaches.

### The Spectrum of Context Reduction

Classical abstractive summarizers (Pointer-Generator, PEGASUS, BART, T5) enable document condensation but lack task-awareness. Instruction-following summarizers (FLAN-T5, LLM-based) are more robust but still fail to preserve functional dependencies across multi-step workflows.

Token-level learned compressors like LLMLingua achieve significant length reductions with minimal single-turn degradation. These methods are query-aware—optimizing relevance to one upcoming question—but not plan-aware: they ignore information needed across multiple future steps.

More targeted approaches attempt task-aware reduction. Self-RAG prunes irrelevant retrieved context conditioned on a query but doesn't generalize to multi-step workflows with tool outputs and plan transitions. Agent Context Optimization compresses interaction histories via natural-language feedback but remains restricted to next-step compression without modeling plan structure. Binary relevance classifiers like Provence prune document segments too coarsely for workflows requiring fine-grained transformations of reasoning chains and state representations.

### Why Pure Summarization Fails for Agents

Standard summarization flattens causal links across steps, removing structural dependencies required for multi-step reasoning. For agent workflows, content often must be rewritten rather than simply removed: global plan structure must be preserved because task dependencies are multi-hop, and instructions drift if not rewritten in sync with evolving context. Effective agent compression therefore requires structured rewriting, task-dependent reduction, and instruction refinement performed jointly—precisely the gap distillation-based compressors fill.

### Instruction Drift and Co-Refinement

A specific failure mode in long-horizon settings is instruction drift: as context accumulates and is modified, original instructions may contradict the current state, reference removed information, or ignore intermediate results. Conditioning compression on upcoming tasks preserves alignment between the agent's plan and evolving state.

An emerging concept is instruction–context co-evolution, where instructions are refined jointly with context selection rather than treated as static inputs. In practice this occurs implicitly when instructions are embedded within and reshaped by compressed context. This joint refinement prevents cascading errors from instruction-context divergence over extended interactions. No prior work in prompt optimization has systematically addressed next-k-task instruction refinement or plan-conditioned instruction rewriting as integrated components of context management.

### Teacher-Student Distillation for Plan-Aware Compression

A large teacher LLM performs high-quality, plan-aware compressions across many workflows; a smaller student is then trained to approximate this behavior via causal language modeling loss. Because the teacher jointly reasons about plan structure, tool dependencies, and instruction fidelity, the resulting compressions preserve multi-hop causal links and state representations that downstream steps depend on.

Large-scale synthetic data generation is a critical enabler. Synthetic workflow systems produce millions of annotated traces spanning document-centric workflows, web navigation, multi-application interactions, and multi-hop QA with retrieval. These include explicit plan structures, multi-hop dependencies, variable noise levels, and rich tool-use patterns (search, browsing, document manipulation, spreadsheet operations, structured extraction), with plans ranging from 5 to 30 steps—providing dense supervision without expensive human annotation.

### Quality Filtering as the Linchpin

Outcome-level agreement between compressed and full-context execution serves as the primary quality gate: only trajectories where the compressed version produces semantically equivalent results are retained. Filtering combines embedding-based semantic similarity (cosine similarity thresholds ~0.92) with LLM-based judging providing binary assessments plus rationales. This prevents mode collapse and catches failures invisible to token-level metrics—e.g., a compressor silently dropping a variable binding needed three steps later. Ablation studies show removing either filter increases failure rates by ~8–12%.

### Meta-Learning the Compression Policy

The teacher's compression policy can be meta-learned through an LLM-driven evolutionary loop. A population of natural-language compression prompts is maintained, each evaluated on independent workflows and scored on success rate, semantic equivalence, compression ratio, and a composite fidelity-compression reward. Evolution proceeds in steady-state, asynchronous fashion. This search over compression strategies discovers when to summarize, rewrite instructions, or preserve content verbatim—yielding inherently plan-aware policies.

### Efficiency and Deployment

Distilled students retain ~97% of teacher performance while reducing inference cost by over an order of magnitude. At inference time, compact compressors add <8% latency per step while reducing input tokens by 35–60%, yielding net cost reductions. The teacher is used only during offline data generation; once distilled, the student is reused across tasks with no further teacher invocation. This asymmetry—expensive offline training, cheap online inference—makes distillation economically viable versus running the teacher at every step or using simpler methods that sacrifice plan-awareness for speed.
