# LLM and Agent Routing

Model routing refers to the intelligent selection of the most suitable LLM from a pool of candidates for a given task or sub-task, exploiting complementary model capabilities while managing computational costs. Routing approaches can be categorized along several dimensions of granularity.

**Single model routing** assigns one model to handle an entire task. Approaches include neural network-based routers trained on performance or cost signals—such as pairwise-comparison aggregation methods and reward-guided label augmentation—as well as graph-based routers and cluster-based selection methods. These have primarily been evaluated on single-turn response benchmarks (e.g., Chatbot Arena, MATH) and struggle with complex agentic workflows where different sub-tasks (web browsing, coding, summarization) demand distinct model capabilities.

**Multi-agent routing** selects different models for different agent roles within a multi-agent system. While this aligns more naturally with the structure of agentic architectures, existing implementations have been limited to relatively simple, single-step reasoning scenarios (e.g., math reasoning and code generation) and depend on large-scale trajectory data to model agent behavior, making them poorly suited for dynamic, multi-turn settings.

**Step-level dynamic routing** operates at the granularity of individual sub-tasks within a complex workflow, selecting the most appropriate LLM before each execution step rather than committing to a fixed assignment. This finer granularity enables adaptation to the varying difficulty and nature of sub-tasks encountered during multi-step agentic workflows. Empirical analysis of such routing reveals that model allocation naturally aligns with sub-task complexity: cognitively demanding planning tasks favor high-capability models, while simpler operational tasks (e.g., file handling) predominantly use cost-efficient smaller models.

A critical characteristic of most established agentic systems is that the mapping from agent roles to LLM backbones is static and human-predefined—for example, hard-coding a web-browser agent to use GPT-4o or a planner agent to use o3-mini. This rigid design lacks adaptability and fails to account for the varying demands of different sub-tasks encountered during execution. Step-level dynamic routing directly addresses this limitation by treating model selection as a decision that can vary at each point in the execution trace.

The relationship between routing granularity and system economics is significant. Single-model routing optimizes globally but sacrifices local fit; multi-agent routing respects role boundaries but remains static within them; step-level routing maximizes adaptability but introduces overhead from repeated selection decisions. The choice among these approaches is itself a trade-off between the precision of model allocation and the complexity of the routing mechanism—a trade-off that mirrors the broader tension among performance, cost, and efficiency in agentic systems.
