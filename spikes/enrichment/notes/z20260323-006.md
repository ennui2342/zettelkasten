# Multi-Agent Reasoning Systems: Formal Foundations, Communication-Computation Tradeoffs, and Scaling Laws

Chain-of-thought (CoT) prompting and its extensions (self-consistency, tree-of-thoughts, stream-of-search) significantly improve LLM reasoning by generating intermediate steps. Post-training increasingly uses reinforcement learning over generated chains. Yet even the strongest CoT-augmented models degrade as problem complexity or context length grows—reflecting structural limits on bounded-depth Transformer computation that formal expressiveness results simultaneously illuminate and bound.

Multi-agent reasoning addresses these limits by distributing long-context tasks across agents handling shorter subtasks. This moves beyond heuristic design toward formally analyzable architectures.

### Formal Framework: Reasoning as a DAG

A multi-agent reasoning system is modeled as a directed acyclic graph where nodes are agent states and edges encode either CoT token generation (within an agent) or inter-agent communication. Key complexity dimensions:

- **Depth**: longest DAG path (proxy for wall-clock time)
- **Width**: number of parallel agents (typically total input size / chunk size)
- **Total size**: aggregate computational work
- **Communication budget**: number of inter-agent message events

Each agent receives an equal input partition processed alongside CoT and communication tokens within its context window. This maps onto document chunking, corpus sharding, browser-agent page sections, and map-reduce pipelines.

### Three Communication Regimes

Representative task families reveal three regimes governing how partitioning interacts with communication:

**Communication-free.** Tasks like key-query retrieval require negligible inter-agent communication—each agent independently checks its chunk, sending only a minimal signal to the manager.

**Beneficial partitioning.** Tasks like state tracking actively benefit from distribution. Agents process local transition subsequences and compose partial results with modest communication, reducing computation depth.

**Communication-intensive.** Multi-hop reasoning, where facts are scattered across chunks, demands substantial inter-agent exchange. Communication budget becomes the binding constraint, and insufficient bandwidth directly limits accuracy. This regime is especially significant because multi-hop reasoning is precisely where single-agent CoT most conspicuously fails—yet the multi-agent alternative introduces coordination costs.

### Scaling Laws and Resource Allocation

A fixed total budget creates a fundamental scale-out vs. scale-up tradeoff. Single-agent performance follows a power-law: bias $b(x) \approx c \cdot x^\alpha$ (binary tasks) or MSE $\sigma^2(x) \approx c' \cdot x^{-2\alpha}$ (continuous tasks), where $\alpha$ is the single-agent scaling exponent.

For a $k$-ary tree of depth $d$ with $n = k^d$ leaves, the budget decomposes as $B \approx n \cdot x + n \cdot c_{\text{edge}} \cdot m$, forcing a direct tradeoff between agent count and per-agent capability.

In the supercritical regime ($\lambda > 1$), the **organization exponent** $\gamma = \log \lambda / \log k$ governs how system performance scales with leaf count. This exponent decreases with higher inter-agent correlation $\rho$ and increases with better communication fidelity $\eta$ and larger fan-in $k$ (with diminishing returns since $M_k'(0) \leq \sqrt{k}$).

**Synergy condition.** Scale-out outperforms scale-up exactly when $\gamma > \alpha$. Optimal per-leaf compute is then $x^* = \alpha B / (\alpha + \gamma)(1 + c_{\text{edge}} m)$. When $\gamma \leq \alpha$, the budget favors fewer, stronger agents. An explicit budget threshold marks the minimum at which organizational benefit first exceeds coordination overhead.

**Continuous tasks** exhibit analogous structure. A star with $n$ leaves achieves MSE $= \rho \sigma^2 + (1-\rho)\sigma^2/n + \sigma_c^2/n$, revealing an irreducible correlation floor. Trees accumulate communication-induced floors across layers. The condition $\gamma_c > \alpha$ again determines scale-out viability.

**Saturation.** Once performance approaches its ceiling (fixed point $b^*$ or correlation/communication floor), additional agents or depth yield diminishing returns. Budget should shift toward improving communication fidelity or reducing shared-failure correlation.

### Tradeoffs and Empirical Validation

The three communication regimes formalize which tasks decompose cleanly and which resist decomposition. Communication-intensive tasks may still benefit from multi-agent treatment when single-agent context is overwhelmed, but only with sufficient communication infrastructure. Empirical validation on pretrained LLMs with synthetic benchmarks confirms that accuracy, communication cost, and token usage track theoretical predictions across all three regimes.

Multi-agent systems can be understood as spatially distributing the reasoning that techniques like tree-of-thoughts and self-consistency distribute across sampling trajectories, with inter-agent communication maintaining coherence. The scaling-law analysis provides concrete guidance: deploy multi-agent architectures when the organization exponent exceeds the single-agent scaling exponent, provision communication proportional to regime demands, and redirect resources from scale-out to parameter improvement once saturation is approached.
