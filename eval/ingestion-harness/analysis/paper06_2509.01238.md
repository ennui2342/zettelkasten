## Within-paper accumulation pattern

A recurring pattern across papers 3, 4, 5, and 6: a note is CREATEd from the first Form draft, then immediately UPDATEd and EDITed by subsequent drafts from the same paper run. The note bloats within a single ingestion before L3 has a chance to fire as a separate decision.

This happens because Form extracts multiple related drafts from one paper, and they all route to the same new note sequentially. The L3 threshold fires only on UPDATE, not on the initial CREATE — so a note can grow from 4000 to 9000+ chars within a single paper run before L3 is aware.

One possible mitigation: run L3 within-paper when a note exceeds the threshold mid-ingestion, not just on the next paper's UPDATE. This would be a pipeline change, not just a prompt change.
